# Academic Workflow Enhancement Project - Claude Code Implementation Brief

## Project Overview
You are enhancing an academic workflow application that helps students and researchers generate outlines, conduct research, and write academic papers. The codebase is in production-ready v1 state with a solid foundation, but requires three critical enhancements to meet user needs.

## Current Architecture Assessment
- **Next.js 14** with TypeScript, Tailwind CSS, App Router
- **Database**: Prisma ORM with SQLite (production-ready schema)
- **Authentication**: NextAuth.js with credentials provider
- **AI Integration**: Claude 3.5 Sonnet (single provider)
- **Testing**: Comprehensive TDD coverage with Vitest, Playwright
- **Workflow**: Complete academic paper generation pipeline (prompt → outline → research → content → export)
- **Export**: PDF/Word generation with BibTeX reference support

## Critical Enhancement Requirements

### 1. Multi-LLM Provider Architecture (HIGH PRIORITY)
**Goal**: Implement intelligent LLM routing with Anthropic Claude + OpenAI GPT-4o

**Implementation Requirements**:
- Create provider abstraction layer in `/lib/ai-providers/`
- Smart routing logic: Claude Sonnet for complex research/analysis, GPT-4o for reviews/validation
- Automatic failover mechanism between providers
- Cost tracking and optimization per provider
- Unified response format across providers

**Files to Enhance**:
- `/pages/api/outline.ts` - Add multi-provider support
- `/pages/api/research.ts` - Implement provider routing
- Create `/lib/ai-providers/anthropic.ts` and `/lib/ai-providers/openai.ts`
- Create `/lib/ai-providers/router.ts` for intelligent routing

**Success Criteria**:
- Users experience seamless AI functionality regardless of which provider is used
- Automatic failover if one provider is unavailable
- Cost optimization through strategic model selection
- Maintain existing API contracts and test compatibility

### 2. Zotero Integration for Reference Management (HIGH PRIORITY)
**Goal**: Bidirectional sync with Zotero libraries for seamless reference management

**Implementation Requirements**:
- Zotero Web API integration for library sync
- Import references from user's Zotero library into research workflow
- Export generated references back to Zotero
- Handle authentication with Zotero (API keys/OAuth)
- Conflict resolution for duplicate references

**Files to Create/Enhance**:
- `/pages/api/zotero/sync.ts` - Bidirectional sync endpoint
- `/pages/api/zotero/import.ts` - Import from Zotero library
- `/pages/api/zotero/export.ts` - Export to Zotero
- `/lib/zotero/` - Zotero API client and utilities
- Update reference management UI in workflow components

**Success Criteria**:
- Users can import existing references from Zotero
- Generated references automatically sync to Zotero library
- Proper handling of citation formats and metadata
- Offline capability with sync when reconnected

### 3. User-Friendly Configuration Management (MEDIUM PRIORITY)
**Goal**: GUI-based settings management for non-technical users

**Implementation Requirements**:
- Settings dashboard UI for API key management
- Secure storage of sensitive configuration (encrypted at rest)
- API key validation and testing within the UI
- Step-by-step setup wizard for first-time users
- Clear documentation of what each setting does

**Files to Create**:
- `/pages/settings.tsx` - Settings dashboard page
- `/pages/api/settings.ts` - Secure settings management
- `/pages/api/test-key.ts` - API key validation endpoint
- `/components/SetupWizard.tsx` - First-time user onboarding
- `/lib/crypto.ts` - Secure storage utilities

**Success Criteria**:
- Non-technical users can configure the app without touching `.env` files
- API keys are validated before saving
- Clear error messages and help text
- Secure handling of sensitive data
