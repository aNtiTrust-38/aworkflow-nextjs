# Settings GUI for Non-Technical Users - Claude Code Implementation Brief

## Project Context
You are implementing a user-friendly settings interface for an academic workflow application. Currently, users must manually edit `.env` files to configure API keys and other settings, which creates a significant barrier for non-technical academic researchers. Your task is to create a complete GUI-based configuration management system.

## Current State Analysis
- Users manually create/edit `.env` files with API keys
- No validation of API keys before use
- No guidance on what each setting does
- Security concerns with plaintext storage
- High barrier to entry for non-technical users

## Implementation Goal
Create a complete settings management system that allows non-technical users to configure the application through a secure, intuitive web interface with validation, testing, and helpful documentation.

## Core Requirements

### 1. Settings Dashboard Page (`/pages/settings.tsx`)
**User Interface Requirements**:
- Clean, academic-focused design using existing Tailwind patterns
- Organized sections: AI Providers, Authentication, Database, Advanced
- Input fields with show/hide toggles for sensitive data
- Test buttons for each API key with status indicators
- Save functionality with success/error feedback
- Help text explaining what each setting does
- Security notice explaining data protection

**Functionality**:
- Load existing settings (masked for security)
- Real-time validation of input formats
- Test API connectivity before saving
- Secure save with encryption
- Clear error messages and guidance

### 2. Backend API Endpoints

#### `/pages/api/settings.ts` - Settings Management
**GET Endpoint**:
- Return masked versions of existing settings (show ••••••••)
- Include status of which settings are configured
- Never expose actual API keys in responses

**POST Endpoint**:
- Validate input formats before saving
- Encrypt sensitive data before storage
- Update environment variables in secure storage
- Return success/error status

#### `/pages/api/test-key.ts` - API Key Validation
**POST Endpoint**:
- Test Anthropic API key with simple request
- Test OpenAI API key with simple request  
- Test database connectivity
- Return validation status without exposing keys
- Rate limit to prevent abuse

### 3. Secure Storage System (`/lib/crypto.ts`)
**Requirements**:
- Encrypt API keys at rest using industry-standard encryption
- Store in secure location (database or encrypted file)
- Provide utilities for encrypt/decrypt operations
- Generate secure random secrets when needed
- Handle key rotation and migration

### 4. Setup Wizard (`/components/SetupWizard.tsx`)
**First-Time User Experience**:
- Welcome screen explaining the setup process
- Step-by-step configuration with progress indicator
- Links to get API keys from providers
- Validation at each step
- Test configuration before completing setup
- Success screen with next steps

### 5. Integration with Existing Auth
**Security Integration**:
- Require authentication to access settings
- Role-based access if multiple users
- Session validation for all settings operations
- Audit logging for security-sensitive changes

### Database Schema Updates
Add to existing Prisma schema:
```prisma
model AppSetting {
  id          String   @id @default(uuid())
  key         String   @unique
  value       String   // Encrypted value
  encrypted   Boolean  @default(true)
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}
```

### Environment Variables to Manage
```typescript
interface AppSettings {
  // AI Providers
  anthropicKey: string;        // ANTHROPIC_API_KEY
  openaiKey?: string;          // OPENAI_API_KEY
  
  // Authentication
  nextauthSecret: string;      // NEXTAUTH_SECRET
  nextauthUrl: string;         // NEXTAUTH_URL
  
  // Database
  databaseUrl?: string;        // DATABASE_URL
  
  // Optional Features
  zoteroApiKey?: string;       // ZOTERO_API_KEY
  enableAnalytics?: boolean;   // ENABLE_ANALYTICS
}
```

## User Experience Flow

### First-Time Setup
1. User visits app, detected as unconfigured
2. Automatic redirect to setup wizard
3. Welcome screen with overview
4. Step 1: Anthropic API key (required)
   - Input field with format validation
   - "Get API Key" link to Anthropic
   - Test button with connection verification
5. Step 2: Optional OpenAI key
   - Explanation of benefits (cost optimization, failover)
   - Skip option clearly available
6. Step 3: Authentication settings
   - Auto-generate secure NextAuth secret
   - Allow manual override for advanced users
7. Step 4: Test configuration
   - Verify all connections
   - Show status of each service
8. Step 5: Complete setup
   - Success message
   - "Start Using App" button

### Ongoing Settings Management
1. Settings accessible via user menu/header
2. Organized dashboard with current status
3. Edit any setting with validation
4. Test connectivity without saving
5. Clear feedback on save success/failure

## Security Requirements

### Data Protection
- Encrypt all API keys using AES-256
- Store encryption key securely (not in code)
- Salt all encrypted values
- Use secure random generation for secrets

### Access Control
- Require authentication for all settings access
- Rate limit API key testing endpoints
- Log all settings changes for audit
- Validate session for every settings operation

### Input Validation
- Format validation for API keys
- SQL injection prevention
- XSS protection in all forms
- CSRF protection on all mutations
